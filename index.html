Digital Implementation Assignment

Root Cause Analysis

Our mid-internship digital implementation assignment: designing an AI/ML-driven RCA Support System for QAPCO.

Pain Point
The Problem We Set Out to Solve

Root Cause Analysis in petrochemical plants is slow, manually intensive, and treated as an isolated, one-off investigation rather than a cumulative organisational learning process. Petrochemical operations involve exothermic reactions, volatile hydrocarbons, and high-pressure systems where slow or shallow RCA carries cost beyond downtime and repairs.

Effects of the Pain Point

1

Recurrence Blindness

The same failure modes repeat because investigations are not connected to a shared historical record.

2

Delayed Corrective Action

Slow data-gathering phases push corrective actions weeks after the incident, extending exposure windows.

3

Cost Impact

Unplanned downtime in high-consequence environments carries costs far beyond the repair itself.

Key Challenges

01

Method Mismatch

Different engineers apply different RCA frameworks with no standardised baseline, leading to inconsistent outputs.

02

Fragmented Data

Process data lives across DCS, SCADA, maintenance logs, and manual records with no unified view.

03

No Historical Cross-checking

Each incident is investigated in isolation. Similar past events never surface as reference context.

Digital Solution
AI/ML-Driven RCA Support System

An AI/ML-driven RCA Support System designed to compress the time between incident occurrence and actionable insight. Not by replacing engineering judgment, but by automating the data-correlation work that humans are structurally slow at performing.

RCA System Core Capabilities

Automated Timeline Reconstruction

Auto-assembles event sequences from DCS, SCADA, and maintenance logs within minutes of incident detection.

Pattern Matching and Historical Cross-reference

Cross-references new incidents against the full historical record to surface similar past events as context.

Probable Cause Ranking

ML classifier ranks probable root causes by confidence score, giving engineers a structured starting point.

Draft Report Generation

Produces a pre-structured RCA draft report so engineers validate rather than write from scratch.

Predictive Risk Flagging

Identifies process conditions that historically precede failures, enabling proactive intervention.

Road to Implementing RCA

Phase 1
Data Audit and Consolidation

Map all existing data sources and assess quality and accessibility.

Phase 2
Historical Data Ingestion

Ingest and label historical incident records to build the training foundation.

Phase 3
Pilot Deployment on a Single Unit

Run the system in parallel with manual RCA on one production unit.

Phase 4
Model Training and Validation

Train and validate the ML classifier against expert-reviewed outcomes.

Phase 5
Integration with Investigation Workflow

Embed the tool into the standard post-incident investigation process.

Phase 6
Feedback Loop and Continuous Learning

Engineers validate or correct AI outputs; corrections retrain the model.

Phase 7
Phased Scale-up

Broader rollout lets the engine draw on a progressively larger and more diverse incident history, improving accuracy over time.

Benefits

FeatureAdvantage
SpeedCompresses the data-gathering phase from days or weeks down to minutes by auto-assembling timelines from DCS, SCADA, and maintenance logs. Engineers start from a validated draft rather than a blank page.
ConsistencyReduces the quality gap between investigators. Instead of outcomes varying based on who is on shift, every investigation starts from the same structured, data-driven baseline.
PreventionCross-referencing new incidents against historical patterns means a similar valve failure from 18 months ago surfaces as context so the same vulnerability is not solved multiple times without addressing the structural cause.
Faster Corrective ActionSince corrective actions cannot be implemented until RCA concludes, speeding up the investigation directly shortens the window of exposure to repeat incidents, which matters most in high-consequence environments like polymerisation or volatile hydrocarbon handling.
